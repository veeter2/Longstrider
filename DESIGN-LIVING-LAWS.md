## 2. Material System: Liquid Glass

### 2.1 Core Principle

**Three visible layers maximum in any viewport region:**
1. **Base** ‚Äî Opaque dark foundation
2. **Glass** ‚Äî Translucent containers with backdrop blur
3. **Content/Overlay** ‚Äî Opaque text surfaces or temporary overlays (modals, toasts)

### 2.2 Glass Treatment Rules

**Standard Glass Surface**:
```css
/* Template - DO NOT modify these values without design approval */
background: rgba(var(--glass-base), 0.6);
backdrop-filter: blur(12px) saturate(1.2);
border: 1px solid rgba(var(--glass-border), 0.15);
```

**Performance Budget**: ‚â§8 backdrop-filters in viewport  
**Priority System**: When limit exceeded, remove lowest priority filters first

**Priority Order** (highest to lowest):
1. HubBar (always visible)
2. Active modal/overlay
3. Contextual sidebar
4. Popovers/tooltips
5. Decorative glass elements

### 2.3 Usage Guidelines

**When to use glass**:
- ‚úÖ Navigation/Chrome (HubBar, sidebars, panels)
- ‚úÖ Overlays (modals, popovers, toasts)
- ‚úÖ Secondary UI elements

**When NOT to use glass**:
- ‚ùå Chat content (readability required)
- ‚ùå Data visualization (contrast critical)
- ‚ùå Text-heavy content
- ‚ùå Form inputs

### 2.4 Anti-Pattern

```tsx
// ‚ùå WRONG - Text directly on translucent glass
<div className="bg-white/10 backdrop-blur-xl">
  <p>This text is hard to read</p>
</div>

// ‚úÖ CORRECT - Text on opaque surface within glass container
<div className="bg-white/10 backdrop-blur-xl p-4">
  <div className="bg-slate-900 p-3 rounded-lg">
    <p>This text is readable</p>
  </div>
</div>
```
